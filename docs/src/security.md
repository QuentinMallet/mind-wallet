# Security Model

`mind-wallet` is a deterministic derivation tool. Anyone who knows the passphrase and profile can derive the same Monero wallet material.

## Passphrase Entropy

The passphrase is the security boundary. `mind-wallet` requires exactly 16 whitespace-separated words. Argon2id slows guessing, but it cannot make a human-memorable low-entropy phrase safe against offline search.

Use 16 independently generated random words with enough entropy to survive dedicated cracking attempts. Avoid quotes, song lyrics, common phrases, modified dictionary words, personal facts, reused passwords, and anything generated by a predictable pattern.

## Deterministic Salt Constraint

The `v1` profile uses a fixed domain string as deterministic salt: `mind-wallet-monero-v1`.

This preserves restore-from-memory behavior without any external salt file, but it also means two users with the same passphrase derive the same wallet. The tool cannot provide per-user uniqueness without adding another secret or stored recovery artifact.

## Terminal and QR Exposure

The mnemonic, private spend key, private view key, and QR bundle are wallet secrets. Treat them like cash-equivalent recovery material.

Risk surfaces include:

- shell history and process listings when using `--passphrase`,
- terminal scrollback and command logs,
- screenshots and screen sharing,
- clipboard managers,
- QR scanners and camera previews,
- remote terminal recording.

## Operational Guidance

- Prefer an offline machine for derivation and restore.
- Clear terminal scrollback after use.
- Do not paste generated secrets into chat, browser tabs, shared notes, or cloud documents.
- Verify official Monero restore compatibility before relying on a profile for funds.
- Record the profile id alongside any recovery plan.
